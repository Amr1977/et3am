## 2. Database Schema Extensions

### 2.1 New Tables

```sql
-- Support Tickets
CREATE TABLE support_tickets (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  type VARCHAR(20) NOT NULL, -- 'bug', 'feature', 'support'
  title VARCHAR(255) NOT NULL,
  description TEXT NOT NULL,
  status VARCHAR(20) DEFAULT 'open', -- 'open', 'in_progress', 'resolved', 'closed'
  priority VARCHAR(20) DEFAULT 'medium', -- 'low', 'medium', 'high'
  assigned_to UUID REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Chat Messages
CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  donation_id UUID REFERENCES donations(id),
  sender_id UUID REFERENCES users(id),
  receiver_id UUID REFERENCES users(id),
  message TEXT NOT NULL,
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- User Reviews
CREATE TABLE user_reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  reviewer_id UUID REFERENCES users(id),
  reviewed_id UUID REFERENCES users(id),
  donation_id UUID REFERENCES donations(id),
  rating INTEGER CHECK (rating >= 1 AND rating <= 5),
  comment TEXT,
  review_type VARCHAR(20) NOT NULL, -- 'donor_to_receiver', 'receiver_to_donor'
  created_at TIMESTAMP DEFAULT NOW()
);

-- Daily Reservation Log (for enforcing daily limit)
CREATE TABLE daily_reservations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  donation_id UUID REFERENCES donations(id),
  reservation_date DATE DEFAULT CURRENT_DATE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Admin Activity Log
CREATE TABLE admin_audit_log (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  admin_id UUID REFERENCES users(id),
  action VARCHAR(100) NOT NULL,
  target_type VARCHAR(50) NOT NULL,
  target_id UUID,
  details JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.2 Modified Tables


## 4. Feature Implementation Details

### 4.1 Meal Browsing & Filtering

**Public users (not logged in):**
- Only see donations with `status = 'available'`
- Hide pickup addresses
- Hide donor information

**Logged-in users:**
- Default: See `status = 'available'`
- Filter selector: `available | reserved | completed | my-donations | my-reservations`
- See addresses for available meals

**Grid view pagination:**
- Page size: 12-24 items
- Cursor-based or offset pagination
- Sort by: nearest, newest, expiring soon

### 4.2 Daily Limit Enforcement

```sql
-- Check before reservation or receiving
SELECT COUNT(*) FROM daily_reservations 
WHERE user_id = $1 AND reservation_date = CURRENT_DATE;
-- Must return 0 to allow any action
```
**Note:** Daily limit applies to BOTH reserving AND receiving - user can only perform 1 total action per day.

### 4.3 Hash Code Delivery System

- Generate 6-character alphanumeric code on reservation
- Store in `donations.hash_code`
- Display to both donor and receiver
- **Valid until delivered or cancelled** by either party - no time expiry
- Verification endpoint checks hash before marking delivered

### 4.4 Real-time Updates (Socket.IO)

**Events to emit:**
- `meal_added` - New donation available
- `meal_reserved` - Reservation made
- `meal_delivered` - Delivery verified
- `reservation_cancelled` - Reservation cancelled
- `new_message` - Chat message received

**Sound effects (frontend):**
- Use Web Audio API or HTML5 Audio
- Different sounds for each event type

### 4.5 Chat System

- Open channel when donation is reserved
- Both parties auto-joined to `donation_{id}` room
- Messages persisted to `chat_messages` table
- Real-time delivery via Socket.IO
- Conversation ends when donation status = `completed` or `cancelled`

### 4.6 Admin Panel

**Metrics dashboard:**
- Total users, active users
- Donations: created, reserved, completed, expired
- Average daily reservations
- User growth chart (line graph)
- Donation status distribution (pie chart)
- Top donating areas (bar chart)

**Management features:**
- View/edit/delete any user
- View/edit/delete any donation
- Resolve support tickets
- System configuration

### 4.7 Support Tickets

**Ticket types:**
- `bug` - Report system issues
- `feature` - Suggest new features
- `support` - General support

**Flow:**
- User creates ticket
- Admin assigns priority
- Admin resolves and responds
- User can add comments

### 4.8 Reputation System

**Metrics tracked:**
- Total donations made (as donor)
- Total meals received (as receiver)
- Aggregate rating (1-5 stars)
- Reputation score calculation

**Rating categories:**
- Donor to Receiver: timeliness, condition
- Receiver to Donor: accuracy, responsiveness
