# Et3am - Daily Limit Clarification

## User Requirement

> "receiver should be able to reserve and receive 1 meal per day"

## Implementation

**Backend** - `backend/src/routes/donations.ts`:

1. **Reserve endpoint** (`POST /:id/reserve`):
   - Checks `dailyReservations.checkTodayAction(userId)` before allowing reservation
   - If user has already reserved today, returns 429 with `messageKey: 'donation.daily_limit_reached'`
   - After successful reservation, logs action to `daily_reservations` table

2. **Complete endpoint** (`POST /:id/complete`):
   - **No daily limit check** - donor can complete at any time
   - Does NOT log any action (receiver already used their daily limit when reserving)

## How It Works

- **Receiver**: Can reserve 1 meal per day. After reserving, they cannot reserve again until the next day.
- **Donor**: Can complete deliveries without any daily limit - the receiver already used their action when reserving.

## Verification

- ✅ Receiver can reserve a meal (uses their 1 daily action)
- ✅ Receiver cannot reserve again same day (limit reached)
- ✅ Donor can complete reservation without daily limit
- ✅ Receiver receives 1 meal per day maximum
